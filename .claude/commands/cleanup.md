## 목표
OpenAI Harness Engineering 의 **doc-gardening** 을 실천한다.
코드와 어긋난 문서, 죽은 코드, 깨진 링크를 찾아 저장소를 최신·정합 상태로 유지한다.

> **주기**: 주 1회 실행 권장. 기계 검증(링크·스키마 드리프트)은 CI 주간 스케줄이 자동 수행하므로,
> 이 스킬은 **의미 수준**(문서 내용 ↔ 실제 코드 동작 불일치, 죽은 코드)에 집중한다.
> 시작 전 `bash scripts/docs-lint.sh && bash scripts/schema-drift.sh` 로 기계 검증을 먼저 돌려라.

## 절차
1. **오래된 문서 탐지**:
   - `docs/` 와 `AGENTS.md` 가 참조하는 경로/파일/명령이 실제로 존재하는가 (깨진 링크 탐지).
   - 문서가 설명하는 구조와 실제 코드 구조가 어긋난 곳.
2. **죽은 코드 탐지**:
   - 참조 없는 클래스/메서드, 미사용 import, 도달 불가 분기.
   - 사용되지 않는 mapper 쿼리, 고아 VO.
3. **주석/TODO 정리**: 해결된 TODO, 오해를 부르는 주석.
4. **일관성**: 네이밍/패키지가 `docs/conventions.md` 와 어긋난 곳.

## 출력·조치
- 발견 사항을 **분류별 목록**으로 먼저 보고한다 (파일:라인, 근거).
- 명백하고 안전한 수정(깨진 링크, 미사용 import, 오탈자)만 바로 적용한다.
- 코드 삭제 등 위험한 변경은 **사용자 승인 후** 진행한다.
- 정리 후 `/review` 를 권한다.

## 하지 말 것
- 동작이 불확실한 코드를 임의로 삭제하지 않는다.
- 생성물(`build/` 등)·`.env` 는 건드리지 않는다 (훅이 막음).
