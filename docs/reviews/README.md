# docs/reviews/ — 코드 리뷰 기록

`/review` 실행 결과가 여기에 쌓인다. **리뷰 이력을 저장소에 남겨** 누가 봐도
"이 변경이 어떤 지적을 받고 어떻게 처리됐는지" 추적할 수 있게 한다.

## 파일 규칙
- 파일명: `{YYYY-MM-DD}_{주제}.md` (예: `2026-07-10_공통코드-엑셀다운로드.md`)
- 같은 주제의 재검토는 **같은 파일 하단에 "재검토" 섹션 추가** (새 파일 금지)

## 리포트 형식
```markdown
# 리뷰: {주제}
- 일시: 2026-07-10 / 검토자: {git user}
- 대상: {브랜치 또는 커밋범위, 예: main 11b9ea3..c7d7e23}
- 결과: 🔴 0 · 🟡 2 · 🟢 3

## 🔴 차단
(없으면 "없음")
## 🟡 권고
- [파일:라인] 내용 → 제안
## 🟢 양호
- ...

## 조치
- 🟡-1: 반영(커밋 abc1234) / 보류(사유)

## 재검토 (있을 때)
- 일시/결과 ...
```
